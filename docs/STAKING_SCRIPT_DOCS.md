## 1. System Architecture Overview

```
 ┌──────────────────────────────────────────────────────────────────────┐
 │                        WHAT YOU CONTROL                              │
 │                                                                      │
 │   scripts/staking-rewards.py          web/index.html                │
 │   (CLI Python script)                 (Browser-based UI)             │
 └─────────────┬────────────────────────────────────┬──────────────────┘
               │                                    │
               │  HTTP POST (GraphQL)               │  HTTP POST (GraphQL via fetch)
               │                                    │
               ▼                                    ▼
 ┌──────────────────────────────────────────────────────────────────────┐
 │                      Enjin Indexer GraphQL API                       │
 │                                                                      │
 │  Auto-generated by Subsquid from schema.graphql                     │
 │  Exposes: poolMemberRewards, eraRewards, nominationPools, etc.      │
 │  Default port: 4350                                                  │
 └─────────────────────────────────┬────────────────────────────────────┘
                                   │
                   ORM (TypeORM + PostgreSQL)
                                   │
               ┌───────────────────▼───────────────────┐
               │           PostgreSQL Database          │
               │                                        │
               │  pool_member_rewards                   │
               │  era_reward                            │
               │  nomination_pool                       │
               │  pool_member                           │
               │  era                                   │
               └───────────────────┬───────────────────┘
                                   │
                          Written by Indexer
                                   │
               ┌───────────────────▼───────────────────┐
               │    Subsquid Processor (indexer/src/)   │
               │                                        │
               │  Streams blockchain events in real-    │
               │  time via Subsquid Archive or RPC.     │
               │  Processes NominationPools events and  │
               │  writes reward records to PostgreSQL.  │
               └───────────────────┬───────────────────┘
                                   │
                           WebSocket (wss://)
                                   │
               ┌───────────────────▼───────────────────┐
               │   Enjin Relaychain Node (RPC Node)     │
               │                                        │
               │  Emits on-chain events every block.    │
               │  The indexer listens and processes     │
               │  NominationPools.EraRewardsProcessed   │
               │  and NominationPools.RewardPaid.       │
               └────────────────────────────────────────┘
```

**In plain English:**
- The blockchain emits events every era when staking rewards are distributed.
- The indexer (this codebase) listens for those events, calculates each pool member's reward share, and writes everything to a PostgreSQL database.
- The GraphQL API sits on top of PostgreSQL and lets you query the pre-calculated data.
- Both the Python script and the web app query that same GraphQL API.

## 2. How the Script Works (staking-rewards.py)

**File location:** `scripts/staking-rewards.py`

### 2.1 Command-line Usage

```bash
# Basic usage
python scripts/staking-rewards.py <address> <start_era> <end_era>

# With custom endpoint
python scripts/staking-rewards.py <address> 100 150 --endpoint https://your-indexer/graphql

# Output as JSON
python scripts/staking-rewards.py <address> 100 150 --json

# Skip the summary section
python scripts/staking-rewards.py <address> 100 150 --no-summary

# Set endpoint via environment variable
INDEXER_ENDPOINT=https://your-indexer/graphql python scripts/staking-rewards.py <address> 100 150
```

### 2.2 Step-by-Step Execution Flow

```
Step 1:  Parse CLI arguments (address, start_era, end_era, --endpoint)
Step 2:  Validate: start_era ≤ end_era
Step 3:  Print header showing address, era range, endpoint
Step 4:  Call fetch_all_rewards()
    Step 4a:  Set offset = 0, page = 1
    Step 4b:  Call graphql_request() with current offset
    Step 4c:  Append returned rows to all_rows list
    Step 4d:  If rows < PAGE_SIZE (1000): stop (no more pages)
    Step 4e:  Otherwise: offset += 1000, page += 1, goto 4b
Step 5:  If no rows returned → print empty state and exit
Step 6:  If --json → dump rows as JSON and exit
Step 7:  Call render_detail():
    Step 7a:  Group rows by pool ID
    Step 7b:  For each pool: print pool header (name, APY)
    Step 7c:  Print column headers
    Step 7d:  For each era row: print era index, dates, points, reward, accumulated, APY
    Step 7e:  Print pool subtotal
Step 8:  Call render_summary():
    Step 8a:  Print per-pool totals table
    Step 8b:  Print grand total
```

| Package | Version | Purpose |

| `requests` | ≥2.28 | HTTP client for GraphQL requests |

Install dependencies:

## 3. How the Web App Works (web/index.html)

**File location:** `web/index.html`

### 3.1 Serving the Web App

The web app is a **single HTML file** with no build step required. Serve it with any static file server:

```bash
# Python (built-in)
cd web && python -m http.server 8080

# Node.js (npx)
cd web && npx serve .

# Or simply open the file directly in a browser:
open web/index.html
```

> ⚠️ **CORS Notice:** If the GraphQL endpoint is on a different domain from where the web app is served, the browser will block the request unless the indexer's GraphQL server sends the appropriate `Access-Control-Allow-Origin` headers. In that case, either host the web app on the same origin as the indexer, or configure the indexer to allow the web app's origin.

### 3.2 Step-by-Step User Flow

```
Step 1:  User fills in the form:
         · Wallet Address (SS58 relaychain format)
         · Start Era (integer)
         · End Era (integer)
         · GraphQL Endpoint URL

Step 2:  User clicks "Query Rewards" (or presses Enter)

Step 3:  runQuery() validates inputs:
         · Address not empty
         · Start/end era are valid integers
         · Start era ≤ end era
         · Endpoint URL not empty

Step 4:  Log drawer opens automatically
         · Displays: address, era range, endpoint
         · Shows each request being made in real-time

Step 5:  fetchAll() paginates through the GraphQL API:
         · Fetches 1000 records per page
         · Each page fetch is logged in the terminal drawer
         · Continues until a page with < 1000 records is returned

Step 6:  Data processing:
         · groupByPool(): groups records by pool ID
         · Computes grand total ENJ per pool

Step 7:  Rendering:
         · renderStats(): 4 summary stat cards (total reward, pools, eras, records)
         · renderSummary(): pool summary table with totals
         · renderPoolBlocks(): per-pool collapsible sections with era-by-era detail tables

Step 8:  Log drawer shows "Done! All results rendered."
```

### 3.3 Web App State Machine

```
          [Initial]
             |
             | user clicks "Query Rewards"
             ▼
          [Loading]
          - spinner shown
          - log drawer opens
          - button disabled
             |
      ┌──────┴──────┐
      | success     | error
      ▼             ▼
```

## 4. The GraphQL API — Requests & Responses

### 4.1 Endpoint

The Enjin Indexer exposes a GraphQL API (auto-generated by Subsquid from `schema.graphql`).

```
URL:     https://your-indexer-graphql-endpoint/graphql
Method:  POST
```

Configure this URL via:
- Python script: `--endpoint` argument or `INDEXER_ENDPOINT` env variable
- Web app: the "GraphQL Endpoint" input field

### 4.2 Request Headers

Both the script and the web app send the same headers:

```http
POST /graphql HTTP/1.1
Content-Type: application/json
Accept: application/json
```

**Python script additionally sets:**
- Default `User-Agent` from the `requests` library (e.g. `python-requests/2.31.0`)
- `Connection: keep-alive`

**Web app additionally sets:**
- Browser default `User-Agent`
- `Origin: <the page's origin>` (important for CORS — the server must allow this)

### 4.3 Request Body

Every request is a JSON-encoded GraphQL query with variables:

```json
{
    "query": "...(see full query below)...",
    "variables": {
        "address":  "5GrwvaEF5zXb26Fz9rcQpDWS57CtERHpNehXCPcNoHGKutQY",
        "startEra": 100,
        "endEra":   150,
        "limit":    1000,
        "offset":   0
    }
}
```

**Full GraphQL query used by both script and web app:**

```graphql
query StakingRewards(
    $address: String!
    $startEra: Int!
    $endEra: Int!
    $limit: Int!
    $offset: Int!
) {
    poolMemberRewards(
        where: {
            member: { account: { id_eq: $address } }
            eraIndex_gte: $startEra
            eraIndex_lte: $endEra
        }
        orderBy: eraIndex_ASC
        limit: $limit
        offset: $offset
    ) {
        id
        eraIndex
        rewards
        accumulatedRewards
        points
        pool {
            id
            name
            apy
            rate
            state
        }
        reward {
            id
            apy
            averageApy
            reinvested
            active
            rate
            era {
                index
                startAt
                endAt
                startBlock
            }
        }
    }
}
```

#### Filter explanation

| Filter | Description |
|---|---|
| `member: { account: { id_eq: $address } }` | Only records for the given wallet address |
| `eraIndex_gte: $startEra` | Only eras ≥ start era |
| `eraIndex_lte: $endEra` | Only eras ≤ end era |

#### Pagination

| Parameter | Value | Description |
|---|---|---|
| `limit` | 1000 | Records per page |
| `offset` | 0, 1000, 2000, ... | Increments by 1000 until a page returns < 1000 records |
| `orderBy` | `eraIndex_ASC` | Sorted oldest era first |

**When multiple pages are needed:**
- Page 1: `{ limit: 1000, offset: 0 }`
- Page 2: `{ limit: 1000, offset: 1000 }`
- Page N: `{ limit: 1000, offset: (N-1) * 1000 }`
- Stop when the returned array has fewer than 1000 items.

### 4.5 BigInt Fields

The following fields are returned as **strings** (not numbers) in the JSON response, because they exceed JavaScript's safe integer range (`2^53 - 1`):

| Field | Raw Example | Human-readable |
|---|---|---|
| `rewards` | `"1234567890000000000"` | `1.234568 ENJ` |
| `accumulatedRewards` | `"98765432100000000000"` | `98.765432 ENJ` |
| `points` | `"500000000000000000000"` | `500.000000 sENJ` |
| `pool.rate` | `"1002000000000000000"` | `1.002 (rate factor)` |
| `reward.reinvested` | `"50000000000000000000"` | `50.0 ENJ` |
| `reward.active` | `"10000000000000000000000"` | `10,000.0 ENJ` |

**To convert to human-readable ENJ:**
```python
# Python
enj_value = Decimal(raw_string) / Decimal(10 ** 18)

# JavaScript (using native BigInt)
const val = BigInt(rawString);
const whole = val / 1_000_000_000_000_000_000n;
const frac = val % 1_000_000_000_000_000_000n;
const display = `${whole}.${frac.toString().padStart(18, '0').slice(0, 6)}`;
```

## 5. Data Calculations Explained

These are **not computed by the script or web app** — they are pre-computed by the indexer and stored in the database. The script/web app simply display them. This section explains where the values in the response came from.

### 5.1 Pool Rate

Stored in `pool.rate` and `reward.rate`. Updated every era.

```
rate = (pool.balance.active × 10^18) / pool.points
```

- `pool.balance.active` = total ENJ actively staked by the pool (from the staking ledger)
- `pool.points` = total sENJ token supply (all members' balances combined)

The rate starts at `1.0 × 10^18` (i.e. `1000000000000000000`) and grows each era as rewards are compounded. A rate of `1002000000000000000` means 1 sENJ is worth approximately 1.002 ENJ.

### 5.2 Member Era Reward

Stored in `poolMemberRewards.rewards`. Calculated by the indexer on every `EraRewardsProcessed` event.

```
totalPoolPoints = (pool.balance.active × 10^18) / pool.rate

memberReward = (memberPoints × eraReward.reinvested) / totalPoolPoints
```

- `memberPoints` = member's sENJ token balance at era time (`poolMemberRewards.points`)
- `eraReward.reinvested` = total ENJ reinvested into the pool this era
- `totalPoolPoints` = total pool points (re-derived from active and rate for precision)

**As a fraction:** `memberReward = (your_share / total_shares) × total_reinvested`

### 5.3 Accumulated Rewards

Stored in `poolMemberRewards.accumulatedRewards`. A running total.

```
accumulatedRewards[era N] = accumulatedRewards[era N-1] + rewards[era N]
```

This resets to zero per pool when calculated fresh; the indexer accumulates it from the member's first era.

### 5.4 Era APY

Stored in `reward.apy`. Calculated per era.

**First era for a pool:**
```
apy = (rate / 10^18) ^ erasPerYear  −  1,   × 100
```

**Subsequent eras:**
```
newBalance      = eraReward.reinvested + previousEra.active
previousBalance = previousEra.active

apy = (newBalance / previousBalance) ^ erasPerYear  −  1,   × 100
```

`erasPerYear` is a chain configuration value (default: 365 for Enjin chains).

### 5.5 Average APY

Stored in `reward.averageApy`. A smoothed rolling average.

The indexer takes up to the last 30 era reward records and:
1. Discards any era where `|eraApy − poolApy| ≥ 50` (outlier filter)
2. Takes up to the first 15 non-outlier eras
3. Computes the mean APY of those eras

This smoothing prevents single high-reward or low-reward eras from distorting the displayed APY.

## 6. Error Handling & Edge Cases

### 6.1 Python Script Errors

| Situation | How it's handled |
|---|---|
| `requests` not installed | Prints install instructions and exits with code 1 |
| Cannot connect to endpoint | Prints connection error with helpful message, exits with code 1 |
| HTTP error (4xx, 5xx) | Prints HTTP status and exits with code 1 |
| GraphQL errors in response | Prints all error messages from `response.errors`, exits with code 1 |
| No records found | Prints a helpful no-results message with possible reasons, exits with code 0 |
| `start_era > end_era` | Validated before request, exits with code 1 |

### 6.2 Web App Errors

| Situation | How it's handled |
|---|---|
| Empty address field | Red error banner shown, query not sent |
| Invalid era numbers | Red error banner shown, query not sent |
| `start_era > end_era` | Red error banner shown, query not sent |
| Network/fetch error | Error banner + log line with error detail |
| GraphQL errors | Error banner + log line with all error messages |
| CORS blocked | Browser error — user must ensure CORS is configured on the server |
| No records found | Empty state message ("No Rewards Found") with explanation |

### 6.3 CORS for the Web App

If both the web app and the indexer run on the same host/port, CORS is not an issue.

If they are on different origins, the indexer's Subsquid GraphQL server must return:
```http
Access-Control-Allow-Origin: *
Access-Control-Allow-Headers: Content-Type, Accept
Access-Control-Allow-Methods: POST, OPTIONS
```

A workaround is to open `web/index.html` directly as a `file://` URL in Chrome/Edge with `--disable-web-security` flag (development only, never in production).

### 6.4 Missing `eraIndex` Field

Some older `PoolMemberRewards` records may have a `null` `eraIndex` (this is a known issue in the indexer, patched by the `backfill-pool-member-rewards-era-index` job). The script and web app handle this gracefully by displaying `—` for null era index values without crashing.

### 7.3 Getting the Indexer GraphQL Endpoint

The Subsquid-powered indexer exposes its GraphQL API on **port 4350** by default when run locally:

```
http://localhost:4350/graphql
```

When deployed, the URL depends on your infrastructure. It is typically the same host as the indexer, on the configured port. Check `docker-compose.yml` or the indexer's environment configuration.

### 7.4 Finding the Right Era Range

If you don't know what era range to use, you can query the indexer for current era info:

```graphql
# Find the latest era
query {
    eras(orderBy: index_DESC, limit: 1) {
        index
        startAt
        endAt
    }
}
```

Then subtract to get recent eras. For example, if the current era is 500, use `startEra: 470, endEra: 500` to see the last 30 eras.

### 7.5 Address Format

The wallet address must be in **SS58 format** for the Enjin Relaychain (prefix `2135`).

- Enjin Relay addresses typically start with `en` or similar prefix depending on the SS58 encoding.
- The indexer stores addresses in the SS58 format it received from the chain.
- If your address is in a different SS58 format (e.g. generic Substrate with prefix `42`), it may not match records in the indexer. Convert it first using [Subscan's address converter](https://www.subscan.io/tools/format_transform) or the `@polkadot/util-crypto` JS library:

```javascript
import { encodeAddress, decodeAddress } from '@polkadot/util-crypto'
const enjinAddress = encodeAddress(decodeAddress(yourAddress), 2135)
```

| `reward.active` | `bigint (string)` | Total ENJ actively staked by the pool at this era (18 decimals) |

| `reward.era.index` | `int` | Era number |
| `reward.era.startAt` | `datetime` | ISO8601 timestamp when the era started (UTC) |
| `reward.era.endAt` | `datetime \| null` | ISO8601 timestamp when the era ended (null if era is still ongoing) |
| `reward.era.startBlock` | `int` | Block number at which this era started |

### 8.2 Decimal Conversion Quick Reference

All amounts in the API use **18 decimal places** (the native precision of ENJ).

| Raw value | Human-readable |
|---|---|
| `"1000000000000000000"` | `1.000000 ENJ` |
| `"1500000000000000000"` | `1.500000 ENJ` |
| `"500000000000000000"` | `0.500000 ENJ` |
| `"1234567890123456789"` | `1.234568 ENJ` |

```python
# Python
from decimal import Decimal
enj = Decimal("1234567890123456789") / Decimal(10**18)
# → Decimal('1.234567890123456789')

# JavaScript
const raw = 1234567890123456789n;
const whole = raw / 10n**18n;  # → 1n
const frac  = raw % 10n**18n;  # → 234567890123456789n
```

### 8.3 Important ID Formats

| Entity | ID Format | Example |
|---|---|---|
| `NominationPool` | `"{poolId}"` | `"3"` |
| `EraReward` | `"{poolId}-{eraIndex}"` | `"3-142"` |
| `PoolMember` | `"{poolId}-{accountId}"` | `"3-5GrwvaEF..."` |
| `PoolMemberRewards` | `"{poolId}-{accountId}-{eraIndex}"` | `"3-5GrwvaEF...-142"` |
| `Era` | `"{eraIndex}"` | `"142"` |
| `Account` | The SS58 address itself | `"5GrwvaEF..."` |

> **Note on era index offset:** Records from the legacy `RewardPaid` chain event (chains before Enjin v1060) store era index as `chainEventEra + 1`. Records from the modern `EraRewardsProcessed` event use `chainEventEra` directly. Both are stored under `eraIndex` transparently — you don't need to handle this difference when querying.

## Appendix A — Running Both Tools Locally

### Complete Local Setup

```bash
# 1. Install Python dependencies
pip install requests

# 2. Note your indexer's GraphQL endpoint (default Subsquid port is 4350)
export INDEXER_ENDPOINT="http://localhost:4350/graphql"

# 3. Find your current era (optional utility query)
curl -s -X POST "$INDEXER_ENDPOINT" \
  -H "Content-Type: application/json" \
  -d '{"query":"{ eras(orderBy: index_DESC, limit: 1) { index startAt endAt } }"}' \
  | python -m json.tool

# 4. Run the Python script
python scripts/staking-rewards.py \
    "enYourAddressHere" \
    100 \
    150 \
    --endpoint "$INDEXER_ENDPOINT"

# 5. Serve the web app
cd web && python -m http.server 8080
# Then open http://localhost:8080 in your browser
```

### Quick JSON Export

```bash
python scripts/staking-rewards.py "enYourAddress..." 100 200 --json > rewards.json
cat rewards.json | python -m json.tool
```
